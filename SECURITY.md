# Security Policy

## Supported Version

Security fixes are applied to the latest release.

## Reporting

Do not open a public issue containing credentials, production configs, private
addresses, logs, screenshots, or customer details. Report the issue privately
to the repository maintainer and provide only the minimum sanitized reproduction
needed to understand it.

## Data Handling

The application processes local text files and does not require network access.
Users are responsible for reviewing generated output and protecting any local
operational data they choose to process.
